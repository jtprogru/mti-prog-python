# QWEN.md — Контекст для работы с проектом

## Обзор проекта

**mti-prog-python** — это коллекция учебных лабораторных работ по программированию на Python. Репозиторий содержит набор независимых скриптов, демонстрирующих базовые возможности языка: работу с типами данных, ввод/вывод, арифметические операции, строки, структуры данных, файловую систему и ООП.

Проект предназначен для образовательных целей и не требует сборки или тестирования. Каждый скрипт является самодостаточным модулем с функцией `main()` и может запускаться напрямую через интерпретатор Python.

---

## Технологии и зависимости

- **Язык:** Python 3.14 (требуется версия ≥3.14)
- **Виртуальное окружение:** `.venv` (включено в репозиторий)
- **Внешние зависимости:** отсутствуют (используется только стандартная библиотека)

## Запуск скриптов

### Базовый запуск

```bash
python <скрипт>.py
```

Примеры:
```bash
python lab1.py
python calculator.py
python diary_app.py
```

### Работа в виртуальном окружении

```bash
# Активация виртуального окружения (macOS/Linux)
source .venv/bin/activate

# Запуск скрипта
python <скрипт>.py

# Деактивация
deactivate
```

## Архитектура и соглашения

### Структура скриптов

Каждый файл следует единому шаблону:

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-

def main():
    # Основная логика
    pass

if __name__ == "__main__":
    main()
```

### Кодировка

Все файлы используют UTF-8 (указано в shebang-комментарии).

### Стиль кода

- Имена переменных: `snake_case`
- Строки документации: краткие комментарии или docstrings
- Обработка ошибок: `try/except` с валидацией ввода
- Интерактивность: `input()` для взаимодействия с пользователем

### Хранение данных

Некоторые скрипты (`diary_app.py`, `library_system.py`) используют JSON-файлы для сохранения состояния между запусками:
- `tasks.json` — задачи ежедневника
- `library_data.json` — данные библиотеки

## Расширение проекта

### Добавление нового скрипта

1. Создайте файл `<имя>.py` в корне репозитория
2. Добавьте shebang и кодировку:
   ```python
   #!/usr/bin/env python3
   # -*- coding: utf-8 -*-
   ```
3. Реализуйте функцию `main()` с логикой
4. Добавьте блок `if __name__ == "__main__": main()`
5. Обновите `README.md`, добавив скрипт в таблицу

### Добавление тестов (опционально)

При необходимости можно добавить тесты:

```bash
mkdir tests
# Создать test_<скрипт>.py с использованием unittest или pytest
```

### Линтинг и форматирование (опционально)

Рекомендуемые инструменты:
- **flake8** — проверка стиля PEP 8
- **black** — автоматическое форматирование
- **mypy** — статическая проверка типов

## Примечания для Qwen Code

- Все скрипты независимы, нет общего состояния или базы данных
- Проект не требует сборки — запуск через `python <файл>.py`
- При модификации кода сохраняйте существующий стиль (shebang, кодировка, `main()`)
- Не добавляйте внешние зависимости без явного запроса пользователя
- При создании новых файлов используйте UTF-8 и `snake_case` для имён
